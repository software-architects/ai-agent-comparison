# Save named configurations

I have a small web app for practicing jazz chords: https://github.com/karin112358/jazz-chord-flashcards. Right now you can adjust the settings to build one custom configuration, but there's no way to keep more than one — as soon as you change the settings, the previous setup is gone.

I'd like to be able to save a configuration under a name and come back to it later. So: give the current settings a name, store it in the browser's local storage, and let me pick one of my saved configurations to load it again. Being able to rename or delete a saved configuration would be nice too.

Take a look at the repo to understand how the configuration works today, then write up a detailed plan for implementing this. Cover what files and components need to change, how the saved configurations should be structured in local storage, and what the UI for naming, saving, loading and managing them should look like. Please save the plan as a Markdown file — I just want the plan for now, not the implementation.
